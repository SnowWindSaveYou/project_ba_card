# 牌桌布局重设计 — 参考炉石传说 v2

> 核心思路：桌游用卡牌做 token 是因为没有屏幕，电子版完全不需要。
> 桌面只留**需要实际打出/交互的卡牌区域**，角色信息全部 HUD 化。

---

## 一、设计原则

1. **桌面极简**：桌面上只放真正的卡牌（连招链、预备区），其余信息用 HUD 表示
2. **炉石式 HUD**：角色头像、体力条、武器/护具状态、充能水晶都是屏幕覆盖 UI
3. **对称镜像**：己方 HUD 在下半屏，对手 HUD 在上半屏
4. **聚焦连招链**：桌面中央留给核心玩法区域

---

## 二、桌面上保留的区域（3D 卡牌）

只有这些区域会有实体 3D 卡牌出现在桌面上：

| 区域 | 位置 | 说明 |
|------|------|------|
| **连招链** | 桌面正中央 Z=0 | 攻防交替打出的卡牌链，回合结束清空 |
| **己方预备区** | Z=-1.3, X=0 | 面朝下暗扣 1 张（类似炉石的奥秘位） |
| **己方牌库** | Z=-1.3, X=2.3 | 右侧竖叠 |
| **对手预备区** | Z=1.3, X=0 | 镜像 |
| **对手牌库** | Z=1.3, X=2.3 | 镜像 |
| **己方手牌** | 相机容器 | 不在桌面上，挂载于相机子节点 |
| **对手手牌** | Z=1.6 | 背面排列，缩小 |

### 3.3 充能水晶（炉石法力水晶风格）

参考炉石法力水晶的视觉设计：

- **满充能**：实心发光菱形 ◆（蓝色发光）
- **已消耗**：空心暗淡菱形 ◇（灰色）
- 每回合重置为 0，充能一张牌获得对应点数
- 显示为 `已用/总产出` 格式

```lua
-- 绘制充能水晶
-- 每颗水晶 = 1 点体能
-- 最大显示 10 颗（超出用数字补充）
for i = 1, maxEnergy do
    if i <= currentEnergy then
        -- 发光实心菱形
        drawCrystal(ctx, x, y, true)
    else
        -- 暗淡空心菱形
        drawCrystal(ctx, x, y, false)
    end
end
```

### 3.4 弃牌堆/放逐区（角标计数器）

不占独立面板空间，作为角色面板上的小角标：

```
[弃牌 12] [放逐 3]   ← 点击可展开查看列表
```
