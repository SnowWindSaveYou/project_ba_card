# 轻拳飞扬 — 卡牌视觉设计规范

> 风格关键词：**碧蓝档案 × Kawaii Pop**
> 明亮柔和、圆润可爱、层次分明、信息清晰

---

## 1. 设计理念

碧蓝档案的 UI 以**纯净的天蓝/白色系 + 几何光晕**著称，搭配动漫角色立绘；
Kawaii Pop 强调**粉彩色板、圆角形状、简洁图标、无攻击性的柔和阴影**。

两者结合，轻拳飞扬的卡牌应当：

- **一眼可读**：攻/防/费/Pitch 四项核心数据不看详情就能辨认
- **层次立体**：多层 Quad 堆叠产生真实的视差深度，不是平面贴图
- **色彩明快**：泡姆泡姆风格的糖果色系，每个信息点一种专属色
- **圆润友好**：所有徽章/框线倾向圆角/圆形，无尖锐棱角

---

## 2. 卡牌结构（多层 Quad 堆叠）

从底到顶，每层之间有 0.01～0.02m 的真实 Y 轴间距：

```
┌──────────────────────────────────────┐
│  Layer 0 — 卡背 (backPlate)          │  深蓝底板，背面朝上时可见
│  Layer 1 — 卡面底板 (cardBody)       │  白色/奶白磨砂质感
│  Layer 2 — 插画区 (artArea)          │  角色立绘或渐变占位
│  Layer 3 — 卡框边饰 (cardFrame)      │  半透明白色/Pitch色描边
│  Layer 4 — 信息徽章 (badges)         │  攻/防/费/Pitch 色块+数字
│  Layer 5 — 卡名条 (nameBar)          │  底部或顶部半透明名称条
└──────────────────────────────────────┘
```

### 当前实现（Phase 1 — 占位色块）

```
CardRoot  scale=(0.63, 0.005, 0.88)
 ├── backPlate    Y=-3.0   深蓝底板
 ├── cardBody     Y= 0.0   奶白底板
 ├── pitchStrip   Y=+2.0   顶部 Pitch 色带
 ├── costBadge    Y=+4.0   左上紫色
 ├── powerBadge   Y=+4.0   左下珊瑚红
 └── defenseBadge Y=+4.0   右下天蓝
```

### 目标实现（Phase 2 — 完整卡面）

```
CardRoot
 ├── backPlate    深蓝 + 花纹纹理（菱形暗纹）
 ├── cardBody     奶白渐变底板（上白下微粉/微蓝）
 ├── artArea      角色立绘纹理（占卡面 60%，居中偏上）
 ├── cardFrame    半透明卡框（Pitch 色圆角描边，内发光）
 ├── costBadge    左上圆形徽章（紫底白字，数字用 Text3D）
 ├── powerBadge   左下圆形徽章（珊瑚红底白字）
 ├── defenseBadge 右下圆形徽章（天蓝底白字）
 ├── pitchGem     右上角 Pitch 宝石（红/黄/蓝 带高光）
 └── nameBar      底部半透明白条（卡名 + 类型图标）
```

## 4. 徽章设计

碧蓝档案 + Kawaii Pop 风格的徽章应为**圆形/胶囊形**，带有：

### 4.1 Cost 徽章（左上）
- 圆形，紫罗兰底色
- 白色粗体数字居中
- 底部一点深紫阴影
- 顶部椭圆高光（白 20% 透明度）
- 用 ★ 或 ◆ 小图标点缀

### 4.2 Power 徽章（左下）
- 圆形，珊瑚红底色
- 白色数字 + ⚔ 小剑图标
- 同样的阴影 + 高光处理

### 4.3 Defense 徽章（右下）
- 圆形，天蓝底色
- 白色数字 + 🛡 小盾图标
- 同样的阴影 + 高光处理

### 4.4 Pitch 宝石（右上）
- 菱形/六边形小宝石
- 红/黄/蓝三色对应 Pitch 值
- 内部渐变 + 中心高光点
- 边缘细白描边

### 4.5 尺寸参考

```
卡牌尺寸: 63 × 88 mm (0.63 × 0.88m 游戏世界)

徽章直径: ~9mm (卡面宽度的 14%)
徽章边距: ~3mm 距卡边
Pitch 宝石: ~7mm
卡名条高度: ~10mm
插画区: ~38 × 50mm (居中)
```

## 5. 卡面排版

```
┌─────────────────────────┐
│ [Cost]            [Gem] │  ← 徽章层 (Y+4)
│                         │
│    ┌───────────────┐    │
│    │               │    │
│    │   角色立绘     │    │  ← 插画层 (Y+1)
│    │   / 渐变占位   │    │
│    │               │    │
│    └───────────────┘    │
│                         │
│  ┌───────────────────┐  │
│  │   ✨ 卡牌名称     │  │  ← 名称条 (Y+3)
│  └───────────────────┘  │
│ [Power]      [Defense]  │  ← 徽章层 (Y+4)
└─────────────────────────┘
        ↑ Pitch 色带 ↑        ← 色带层 (Y+2)
        (顶部横条/底部渐变)
```

## 6. 卡背设计

碧蓝档案风格的深色卡背：

- **底色**：深海蓝 (30, 40, 75)
- **暗纹**：45° 菱形网格，颜色略亮 (45, 55, 90)
- **中心图标**：游戏 Logo（简化版，白色 15% 透明度）
- **边框**：细白线描边，圆角 3mm
- **四角装饰**：小星星或花纹图案

## 7. 动效设计（与视觉配合）

| 动效 | 视觉配合 |
|------|---------|
| 悬停上浮 | 徽章层 Y 偏移额外增大，增强视差分离感 |
| 出牌抛出 | 弧线轨迹 + 卡面发光描边（Pitch色） |
| 受伤 | 卡面闪烁红色 + 数字弹出 |
| 加防 | 防御徽章脉冲放大一下 |

## 8. 实现路线

### Phase 1（当前）— 占位色块 ✅
- 多层 Plane 纯色材质
- 基础 Pitch 色带 + 攻/防/费 色块
- 验证视差效果和层级结构

### Phase 2 — Text3D 数字
- 在 costBadge/powerBadge/defenseBadge 上添加 Text3D 子节点
- 白色粗体数字，fontSize 适配徽章大小
- 卡名条添加 Text3D 显示卡名

### Phase 3 — 纹理素材
- AI 生成角色立绘（碧蓝档案画风）
- 制作卡框纹理（半透明 PNG，圆角描边 + 内发光）
- 卡背暗纹纹理
- 徽章纹理（圆形带高光）

### Phase 4 — 完整卡面
- NanoVG Render-to-Texture 或直接纹理贴图
- 完整卡面渲染：立绘 + 卡框 + 徽章 + 名称
- 悬停时额外的视差增强
