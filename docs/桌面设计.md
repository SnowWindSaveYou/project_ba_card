## 1. 总览与核心概念

### 1.1 效果流程（一句话）

按空格触发 → 金色方块从左到右横扫桌面 → 金色从中心退去露出红蓝分区 → 双方分数对向冲撞碰撞 → 胜方半场变金 → 攻击射线 + 火焰爆炸 → 全部消散回归桌面。

### 1.2 核心机制

| 机制 | 说明 |
|------|------|
| **双层网格** | 桌面被 18×28 的小格子覆盖，每格有底层/顶层两个颜色通道，通过 cover/reveal 模式实现无缝色彩过渡 |
| **延迟噪声** | 每格有独立的 delay + 随机噪声，产生参差不齐的锯齿波前 |
| **波环模型** | SCORES_CLASH 阶段，波环从边缘向中心收缩，格子受波环影响产生高度起伏 |
| **方块粒子** | 所有爆发/飞溅效果统一使用旋转方块粒子 |
| **漫画式攻击** | 金色点阵射线 → 火焰爆炸 → 漫画星爆 "-2" |

## 2. 状态机与时序

### 2.1 七阶段状态机

```
IDLE(0) ──SPACE──► BLOCKS_ENTER(1) ──► SCORES_REVEAL(2) ──► SCORES_CLASH(3)
                                                                    │
  IDLE ◄──── DISSOLVE(6) ◄──── ATTACK_EXEC(5) ◄──── WINNER_GLOW(4) ◄┘
```

### 2.2 各阶段持续时间

| # | 状态 | 持续时间(s) | 描述 |
|---|------|------------|------|
| 1 | BLOCKS_ENTER | 2.0 | 金色方块从左到右横扫覆盖桌面 |
| 2 | SCORES_REVEAL | 1.8 | 金色从中心向外退去，露出红蓝分区 |
| 3 | SCORES_CLASH | 2.2 | 分数从两端对冲碰撞 + 波环收缩 |
| 4 | WINNER_GLOW | 1.5 | 胜方（上半）变金，分数更新 |
| 5 | ATTACK_EXEC | 2.2 | 攻击射线 + 火焰 + 漫画星爆 |
| 6 | DISSOLVE | 2.0 | 从中心向外清除所有颜色 |

### 2.3 progress 计算

每帧更新：
```
timer += deltaTime
progress = min(1.0, timer / duration)
```

当 `progress >= 1.0` 时，触发初始化逻辑并切换到下一状态（重置 timer 和 progress）。

## 4. 卡桌布局

### 4.1 尺寸计算

```
margin = screenHeight × 0.07
tableH = screenHeight - margin × 2
tableW = tableH × 0.58              -- 竖长牌桌
tableX = (screenWidth - tableW) / 2 -- 水平居中
tableY = margin
arcH   = tableH × 0.07              -- 顶部拱形高度
```

### 4.2 外形

桌子底部为圆角矩形，顶部为贝塞尔弧线（模拟真实牌桌的弧形短边）。两侧各有一个矩形凹槽装饰。

背景包含：
- 蓝色渐变天空 `rgb(95,155,235)` → `rgb(135,185,250)`
- 6 朵飘动云朵（不同速度/方向）
- 桌面内棋盘格底纹（18px 格子，alpha=0.08）

## 5. 双层网格系统

### 5.1 网格参数

```
GRID_COLS = 18
GRID_ROWS = 28
cellW = tableW / GRID_COLS
cellH = tableH / GRID_ROWS
```

### 5.2 每格数据结构

```
Cell {
    -- 底层颜色
    r, g, b, a: float

    -- 顶层颜色
    pr, pg, pb, pa: float

    -- 过渡控制
    fade: float      -- 0 → 1，表示过渡进度
    coverMode: bool  -- true = 顶层渐显（cover），false = 顶层渐隐（reveal）
    delay: float     -- 过渡开始前的等待时间（秒）
    speed: float     -- 过渡速度（fade 每秒增量）

    -- 视觉效果
    elev: float      -- 当前高度偏移（像素），用于砖块凸起
    elevTgt: float   -- 目标高度
    glowRand: float  -- 0~1 随机值，控制波前闪亮
}
```

### 5.3 两种过渡模式

#### Cover 模式（coverMode = true）

新颜色从**顶层**盖下来。底层保持不变，顶层从透明→不透明。

```
topAlpha = pa × min(1, fade)
```

用途：金色横扫（BLOCKS_ENTER）、胜方变金（WINNER_GLOW）

#### Reveal 模式（coverMode = false）

旧颜色在**顶层**渐隐，露出底层新颜色。

```
topAlpha = pa × max(0, 1 - fade)
```

用途：红蓝揭露（SCORES_REVEAL）、清除消散（DISSOLVE）

### 5.4 Bake 规则 ⭐ 关键

**当过渡完成（fade >= 1.0）或发起新过渡时，必须 bake**：

- **Cover 完成时**：将顶层颜色复制到底层 → `(r,g,b,a) = (pr,pg,pb,pa)`，然后清除顶层
- **发起新 Reveal 时**：如果前一次是 Cover 模式 → 先 bake 顶层到底层
- **发起新 Cover 时**：如果前一次 Cover 已完成 → 先 bake

不 bake 会导致颜色叠加错误。

### 5.5 延迟噪声

每格的 `delay` 基于某个基准值（如距中心距离、列归一化位置）乘以最大延迟时间，再叠加 ±18% 的随机噪声：

```
noise = (random() - 0.5) × maxDelay × 0.18
delay = max(0, baseDelay + noise)
```

### 5.6 glowRand 闪亮效果

在波前附近（`fade > 0 && fade < 0.85 && delay <= 0`），如果 `glowRand > 0.55`，则在格子上叠加一层白色半透明覆盖：

```
glowStrength = glowRand × (1 - fade) × 0.45
overlayColor = rgba(1, 1, 0.9, glowStrength)
```

### 5.7 砖块凸起效果

当 `elev > 1.0` 时，格子渲染为凸起砖块：

- 底色填满整格（缝隙露出此色）
- 顶面内缩 `gap = 0.6 + elevIntensity × 0.4` 像素
- 顶面颜色在原色基础上叠加白色提亮 `+= elevIntensity × 0.35`
- 轻微向上偏移 `-= elevIntensity × 0.6` 模拟高度

高度过渡每帧平滑插值：`elev += (elevTgt - elev) × min(1, 8 × dt)`

## 6. 各阶段详细设计

### 6.1 BLOCKS_ENTER — 金色横扫

**网格操作**：`GridSweepLeftToRight(GOLD, alpha=0.9, duration=1.4, speed=6.0)`

- 模式：Cover（新金色从顶层盖下来）
- 延迟：`delay = colNorm × 1.4 + noise`（列归一化 × 1.4s）
- 前缘位置：`frontNorm = min(1.0, elapsed / 1.4)`

**叠加效果**：

1. **金色区域水印**（scissor 到已覆盖区域内）：
   - "伤害" / "结算" 两行大字（52px，alpha=0.22，深金色）
   - 四个角各一个骰子图标（22px 圆角方框 + 3 点对角排列，alpha=0.15）
   - 6 个卡牌位置剪影轮廓

2. **粉色横条**（追踪金色前缘）：
   - 位置：`tableY + tableH × 0.18`
   - 宽度随扫过进度增长，从左到右渐变 `rgba(0.9,0.2,0.5, 0→0.55)`
   - 右端亮边高光 30px

3. **前缘亮条**：
   - 后方 40px 暖光渐变 `rgba(1,0.9,0.5, 0→0.3)`
   - 中心 4px 亮线 `rgba(1,1,0.92, 0.6)`
   - 前方 20px 微光

4. **粒子**：每帧在前缘位置生成 3~5 个金色/白色方块粒子

### 6.2 SCORES_REVEAL — 中心揭露红蓝

**网格操作**：`GridRevealRedBlue(maxDelay=0.5, speed=5.0)`

- 模式：Reveal（旧金色在顶层渐隐，底层露出红/蓝）
- 上半（row 1 ~ ROWS/2）→ 红色 `(0.82, 0.18, 0.28, 0.85)`
- 下半（row ROWS/2+1 ~ ROWS）→ 蓝色 `(0.25, 0.32, 0.72, 0.85)`
- 延迟：`delay = distToCenter × 0.5 + noise`

**粒子**：沿扩展环前缘生成金色方块粒子（径向运动）

### 6.3 SCORES_CLASH — 分数碰撞

这是最复杂的阶段，包含分数对冲、碰撞、波环三个子系统。

#### 6.3.1 分数对冲时序

| 进度 | 阶段 | 数字位移 | 效果 |
|------|------|---------|------|
| 0% ~ 25% | 缓慢蓄力 | easeInOutQuad(p/0.25) × 0.6 | 速度线渐显（alpha 0→0.4） |
| 25% ~ 35% | 急加速冲刺 | 0.6 + easeOutCubic(sprint) × 0.4 | 速度线增强到 1.0 |
| 35% | **碰撞瞬间** | 1.0 | 全屏震动 8.0、VS 缩放 3.0×、数字缩放 1.45×、水平闪光、数字倾斜 ±0.18 rad |
| 35% ~ 100% | 弹性回弹 | sin(bounceP × π×3) × e^(-bounceP×3.5) | 衰减振荡 |

#### 6.3.2 碰撞瞬间粒子

```
SpawnBurst(中心, 25个, PINK, speed=350, size=5~16, life=0.7)
SpawnBurst(中心, 25个, CYAN, speed=350, size=5~16, life=0.7)
SpawnBurst(中心, 12个, GOLD, speed=400, size=8~20, life=0.9)
SpawnBurst(中心, 8个, WHITE, speed=200, size=3~8, life=0.4)
SpawnDirectional(中心, 10个, PINK, 向左, spread=0.3, speed=350)
SpawnDirectional(中心, 10个, CYAN, 向右, spread=0.3, speed=350)
```

#### 6.3.3 水平闪光线

碰撞瞬间 `clashFlash = 1.0`，每帧衰减 `× (1 - 8 × dt)`

渲染三层：
1. 主体：跨桌面宽度，红→蓝渐变，高度 `4 + flash × 12`
2. 白色核心：3px 高
3. 辉光扩散：三倍高度，alpha=0.15

#### 6.3.4 碰撞星形爆炸

8 条射线从中心向外，长度随时间脉动：`0.7 + sin(time×6+i) × 0.3`
中心径向渐变发光圆

#### 6.3.5 速度线

碰撞前，在两个数字附近各画 6 条速度线（垂直短线段），表现加速感。

#### 6.3.6 波环模型 ⭐

**蓄力阶段（碰撞前）**：

波环从场地边界向中心收缩，与分数运动同步。

```
waveFront = 1.05 × (1 - moveProgress)    // 从边界→中心
peakElev = 5                              // 波环经过时的峰值高度
restElev = 3.5                            // 波前扫过后的安息高度
waveWidth = 0.15                          // 波环宽度

对于每个格子：
  d = distToCenter(row, col)  // 归一化距离 0~1
  
  if d >= waveFront:
    passedDist = d - waveFront
    if passedDist < waveWidth:
      // 波环正在经过：接近峰值
      elev = peakElev × (1 - passedDist/waveWidth × 0.1)
    else:
      // 波环已离开：从峰值→安息高度（easeOut）
      settleTime = min(1, (passedDist - waveWidth) / 0.8)
      rest = max(0, (1-d)^1.0 × restElev + noise)
      elev = lerp(peakElev, rest, 1-(1-settleTime)^2)
```

每格的噪声是预计算的确定性伪随机：
```
noise = sin(row×0.9 + col×1.3) × 0.7
      + cos(row×1.7 - col×0.6) × 0.4
      + (random()-0.5) × 0.5
```

**碰撞后余韵**：

全部高度按 `(1 - decayP)^2.5` 曲线衰减到 0。

#### 6.3.7 进入下一阶段的条件

`progress >= 1.0` **且** 波环余韵完全消散（所有格子 `elev < 0.1`）。

**胜方变金的初始化**：

上半区（row 1 ~ ROWS/2）金色从顶层盖下来，延迟按**斜角**排列：
```
diag = (midRow - row) + (col - 1)
baseDelay = (diag / maxDiag) × 0.5s + noise
```

### 6.4 WINNER_GLOW — 胜方高亮

- `winnerAlpha = easeOutCubic(progress)`
- 分数更新：scoreB 从 16 → 5（`lerp(16, 5, easeOutCubic(min(1, progress×1.5)))`）
- 数字归位：scoreAY、scoreBY 从 1.0 → 0

**皇冠水印**（也在 ATTACK_EXEC 阶段持续显示）：

- 区域：裁剪到桌面上半部分
- 布局：旋转坐标系 -0.35 rad（约 -20°），在旋转后的坐标系中密铺
- 间距：X=65px，Y=55px，交错行偏移半个间距
- 滚动：沿倾斜方向以 18 px/s 匀速移动（在旋转坐标系中仅 X 轴偏移）
- 透明度：`winnerAlpha × 0.14`
- 图标尺寸：28px

皇冠图标：NanoVG 路径绘制
```
底部横线 → 右上升 → 右尖(-4) → 右谷(2) → 中尖(-14, 最高) → 左谷(2) → 左尖(-4) → 回底
三颗装饰圆珠：x=[-10, 0, 10]，y根据位置调整
```

### 6.5 ATTACK_EXEC — 攻击效果

#### 6.5.1 时序

| 进度 | 阶段 |
|------|------|
| 0% ~ 35% | 射线从胜方角延伸到败方角 |
| 35% | 撞击瞬间：火焰 + 星爆 + 粒子 + 震动 |
| 35% ~ 100% | 射线渐隐 |

#### 6.5.2 射线路径

```
起点：(tableX + tableW × 0.75, tableY + tableH × 0.25)  -- 胜方区域右上
终点：(tableX + tableW × 0.25, tableY + tableH × 0.78)  -- 败方区域左下
```

#### 6.5.3 金色点阵射线 (DrawAttackLine)

射线由间距 9px 的小方块组成：

```
对于沿射线方向每 9px 一个方块：
  tailFade = min(1, (headP - currentP) × 6)     // 尾部渐隐
  headGlow = 1 - min(1, (headP - currentP) × 3) // 头部发光更亮
  alpha = (0.5 + headGlow × 0.5) × tailFade
  size = baseSize + (每3个方块交替 +2px)
  rotation = time × 4 + index × 0.7              // 持续旋转

  // 头部附近有外发光层（1.2× 大的半透明方块）
```

射线头部：青色能量球
```
ballRadius = 8 + sin(time × 12) × 2
外发光：径向渐变，半径 = ballR × 2.5，rgba(0.3, 0.9, 1, 0.35)
核心：径向渐变，白芯 → 青色边缘
```

#### 6.5.4 火焰爆炸 (DrawFireExplosion)

撞击点三层径向渐变：
```
内核：R=35px，rgba(1, 0.6, 0.1, alpha×0.7)   -- 橙黄
中层：R=55px，rgba(1, 0.35, 0.05, alpha×0.7)  -- 橙红
外层：R=75px，rgba(0.9, 0.15, 0.05, alpha×0.7) -- 暗红
```

10 条火苗射线：
```
长度 = (30 + sin(time×8+i×1.7) × 15) × alpha
颜色 = rgba(1, 0.7, 0.2, alpha × 0.6)
宽度 = 2.5
```

`fireAlpha` 衰减：`× (1 - 4 × dt)`

#### 6.5.5 漫画星爆 "-2" (DrawDamageStarburst)

位置：撞击点偏右上 `(+30, -35)`

红色星爆背景：12 个尖角，外径 38px，内径 20px
```
红色填充：rgba(0.9, 0.15, 0.1, alpha × 0.9)
白色描边：2px
```

"-2" 白色文字（28px），带黑色阴影偏移 (1, 2)

`dmgScale` 初始 1.5，平滑回落到 1.0：`+= (1.0 - dmgScale) × min(1, 5 × dt)`

`dmgAlpha` 衰减：`× (1 - 2.5 × dt)`

#### 6.5.6 撞击粒子

```
SpawnBurst(撞击点, 15个, RED, speed=250, size=5~14, life=0.6)
SpawnBurst(撞击点, 10个, GOLD, speed=200, size=4~10, life=0.5)
SpawnBurst(撞击点, 5个, WHITE, speed=150, size=3~8, life=0.3)
```

### 6.6 DISSOLVE — 消散

**网格操作**：`GridClearFromCenter(maxDelay=0.7, speed=5.0)`

- 模式：Reveal（当前颜色移到顶层渐隐，底层为透明）
- 延迟：`delay = distToCenter × 0.7 + noise`

**残留效果衰减**：
```
winnerAlpha -= dt × 3
fireAlpha -= dt × 4
dmgAlpha -= dt × 3
```

**粒子**：沿清除前缘生成红色（上半）/蓝色（下半）方块粒子。

## 7. 粒子系统

### 7.1 粒子数据结构

```
Particle {
    x, y: float         -- 位置
    vx, vy: float       -- 速度
    size: float          -- 方块边长
    color: (r, g, b)     -- 颜色
    rot: float           -- 旋转角度（度）
    rotSpd: float        -- 旋转速度（度/秒），±300 随机
    life: float          -- 剩余生命
    maxLife: float       -- 最大生命
    glow: bool           -- 是否有外发光层
}
```

### 7.2 更新逻辑

```
每帧：
  x += vx × dt
  y += vy × dt
  vy += 120 × dt      // 重力
  vx *= (1 - 0.5 × dt) // 空气阻力
  rot += rotSpd × dt
  life -= dt
  if life <= 0 then 删除
```

### 7.3 渲染

```
alpha = min(1.0, life / (maxLife × 0.3))  // 前 30% 生命渐入

// glow 外发光层：2× 大小，alpha × 0.2
// 主体方块：1× 大小，alpha × 0.9
// 白色描边：1px，alpha × 0.3
```

### 7.4 生成函数

- `SpawnBurst(cx, cy, count, color, speed, sizeMin, sizeMax, life)` — 径向爆发
- `SpawnDirectional(cx, cy, count, color, dirX, dirY, spread, speed)` — 定向喷射

上限 500 个粒子。

## 8. 缓动函数

```
easeOutCubic(t)  = 1 - (1 - t)^3
easeInOutQuad(t) = t < 0.5 ? 2t² : 1 - (-2t + 2)² / 2
easeOutBack(t)   = 1 + 2.70158 × (t-1)³ + 1.70158 × (t-1)²
lerp(a, b, t)    = a + (b - a) × clamp(t, 0, 1)
```

---

## 9. 屏幕震动

```
每帧衰减：screenShake *= (1 - 6 × dt)

渲染偏移：
  shakeX = sin(time × 45) × screenShake
  shakeY = cos(time × 40) × screenShake × 0.7
```

触发时机：
- SCORES_CLASH 碰撞瞬间：`screenShake = 8.0`
- ATTACK_EXEC 撞击瞬间：`screenShake = 7.0`

## 10. Unity 移植建议

### 10.1 网格系统

- 使用 18×28 个 `Image` 或 `SpriteRenderer`，挂载在一个 Grid 父对象下
- 每格一个 `GridCell` 组件，包含双层颜色和过渡参数
- Cover/Reveal 通过 `CanvasGroup.alpha` 或直接设置 `Image.color.a`

### 10.2 粒子

- 使用 Unity 的 `ParticleSystem`，或自定义对象池 + `Transform` 旋转
- 方块粒子可用带旋转的 quad sprite

### 10.3 渲染叠加效果

- 分数数字：`TextMeshPro`，用 `RectTransform` 控制位置，`DOTween` 控制碰撞动画
- 射线：`LineRenderer` 或程序化 mesh
- 火焰/星爆：多层 `Image` + `RadialGradient` shader
- 闪光线：全屏宽度 `Image` + 渐变材质

### 10.4 缓动

- 使用 `DOTween` 或 `LeanTween`
- 自定义 `AnimationCurve` 模拟 easeOutCubic 等

### 10.5 波环

- 每格高度映射到 `localPosition.y` 或 `localScale`
- 可选：用 shader 的 vertex displacement 替代逐格处理

### 10.6 其他 Lua 引擎

- Love2D：使用 `love.graphics` 矢量绘制 API（与 NanoVG 接近）
- Defold：使用 GUI 系统或自定义渲染脚本
- Cocos2d-x (Lua)：使用 `DrawNode` 或 custom shader
